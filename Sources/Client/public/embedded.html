Embedded Swift

Swift on microcontrollers — bare metal firmware for IoT devices.

Why Embedded Swift?

Swift's safety guarantees — optionals, value types, strong typing — make it an exceptional language for firmware where bugs can mean hardware failures. Embedded Swift strips away the runtime overhead, enabling Swift to run directly on resource-constrained microcontrollers.

What I Build

I develop firmware for ESP32-C3 microcontrollers using Embedded Swift, targeting real-world IoT applications. My work includes smart irrigation controllers that integrate with Apple Home via the Matter protocol, enabling seamless smart home control.

  • Bare-metal GPIO, timer, and peripheral drivers in Swift
  • Matter protocol stack integration for HomeKit compatibility
  • Low-power design patterns for battery-operated sensors
  • Custom build toolchains with Swift 6.2 and CMake

Architecture Approach

Embedded firmware benefits enormously from Swift's protocol-oriented design. I structure firmware with clean abstractions between hardware interfaces and application logic, making the code testable and portable across different MCU targets.
